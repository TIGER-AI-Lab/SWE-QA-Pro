# Stage 1: Supervised Fine-Tuning (SFT) for SWE-QA-Pro 8B

The first stage of the SWE-QA-Pro training recipe. We fine-tune Qwen3-8B on agentic repository-understanding trajectories generated by Claude Sonnet 4.5, using [ms-swift](https://github.com/modelscope/ms-swift) with the `hermes` agent template.

## 1. Environment Installation

```bash
# Install uv (if not installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

cd path/to/SWE-QA-Pro/train

# Create the env (a folder ./ms-swift) and activate it
uv venv --python 3.10 ms-swift
source ms-swift/bin/activate

uv pip install ms-swift==3.12.0 --torch-backend=auto
uv pip install deepspeed wandb
uv pip install flash-attn==2.5.8 --no-build-isolation --no-binary flash-attn
```

## 2. Data Preparation

Download the trajectories from [`TIGER-Lab/SWE-QA-Pro-SFT-Trajectories`](https://huggingface.co/datasets/TIGER-Lab/SWE-QA-Pro-SFT-Trajectories) into `dataset/`:

```bash
cd path/to/SWE-QA-Pro/train/SFT
hf download TIGER-Lab/SWE-QA-Pro-SFT-Trajectories \
  --repo-type dataset --local-dir dataset --force-download
```

## 3. Training

Before running:

- Authenticate Weights & Biases: set `WANDB_API_KEY` in the script, or run `wandb login`. Leave `WANDB_API_KEY` empty to disable reporting.
- Set `--nproc_per_node` to your GPU count (defaults to 8).

```bash
bash scripts/train_sweqapro_8B.sh
```
